# Hot Trend Lab Release Calendar

A column works only if it can be refreshed. The table below prioritizes topics with public sources, repeating cadence, and high reader interest.

| Cadence | Article | Notebook | Why publish |
|---|---|---|---|
| Weekly | arXiv AI category pulse | `01_arxiv_category_pulse.ipynb` | strongest academic and AI-builder hook |
| Weekly | AI agent / coding-agent research pulse | `02_arxiv_agent_research_pulse.ipynb` | agentic AI remains a fast-moving topic |
| Weekly | Hugging Face open-model pulse | `03_huggingface_open_model_pulse.ipynb` | open-model releases are frequent and shareable |
| Weekly | GitHub AI-agent star velocity | `04_github_ai_agent_star_velocity.ipynb` | developer attention moves quickly |
| Monthly | Wikimedia attention and hype decay | `05_wikipedia_attention_hype_decay.ipynb` | public-interest comparison without financial advice |
| Event-driven | crypto and stablecoin liquidity pulse | `06_crypto_stablecoin_liquidity_pulse.ipynb` | publish only after a large residual event |
| Earnings / macro weeks | AI infrastructure market pulse | `07_ai_infrastructure_market_pulse.ipynb` | strong traffic but requires cautious language |

## Publishing rule

Every article should include:

1. a source table;
2. a decomposition summary table;
3. a top residual events table;
4. a short caveat paragraph;
5. a notebook link.
