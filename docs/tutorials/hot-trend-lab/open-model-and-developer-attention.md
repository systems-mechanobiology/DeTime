# Open Models and Developer Attention

This section tracks two public proxies for technical momentum:

1. Hugging Face model metadata snapshots;
2. GitHub repository stargazer timestamps and star velocity.

Neither source proves model quality or production usage. They measure public attention and adoption proxies.

## Hugging Face model pulse

| Component | Interpretation |
|---|---|
| `trend` | durable download momentum across repeated snapshots |
| `season` | release-week or weekday cadence |
| `residual` | launch spike, benchmark news, controversy, or model replacement |

Notebook:

`examples/notebooks/hot_trends/03_huggingface_open_model_pulse.ipynb`

## GitHub developer pulse

| Component | Interpretation |
|---|---|
| `trend` | sustained developer interest |
| `season` | weekly open-source activity rhythm |
| `residual` | launch, funding announcement, Hacker News burst, or demo release |

Notebook:

`examples/notebooks/hot_trends/04_github_ai_agent_star_velocity.ipynb`

## Editorial rule

Use these sources as attention proxies. Do not write that a repository is technically superior just because its residual shock is large.
